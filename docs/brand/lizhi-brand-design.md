**版本**：v1.0.0

## 0. 设计决策摘要

狸知知识库的品牌核心矛盾在于：**极致的工程安全感 × 安静的猫系陪伴感**。解决这个矛盾的设计策略是「暗色终端美学 + 暖色情感锚点」—— 整体 UI 呈现像 IDE 一样精密冷静的暗色工程界面，但在关键交互点上以「暖爪色」注入猫系品牌的温度。

| 维度 | 决策 |
|------|------|
| 主调性 | 暗色工程终端 — 精密、克制、可验证 |
| 情感锚点 | 暖爪色 `#d4a574` — 猫系温度的唯一出口 |
| 字体系统 | Inter（UI）+ 编程字体（代码区）— 几何精确 |
| 设计参考 | Linear 的暗色深度 + Supabase 的终端气质 + Cursor 的三字体分层 |
| 猫系表达 | Logo 几何猫 + 暖爪色微交互，绝不低幼卡通化 |

## 第二部分：视觉识别（VI）

### 2.1 Logo 系统

**Logo 概念：巢中眼**

几何猫的面孔由三组元素构成：
- **猫耳** → 顶部两个等腰三角形（45° 尖角），象征警觉
- **节点眼** → 两个菱形节点（知识图谱节点），象征知识网络
- **盾形巢** → 底部六边形基座，象征加密保护
- **三节点连线** → 眼下三道连接线，象征双链 / 图谱

```
       ▲           ▲        ← 猫耳（警觉）
      / \         / \
     /   \       /   \
    /     \     /     \
   /       ◇───◇       \    ← 节点眼（知识）
  /         \ /         \
  \    ╱──── ◇ ────╲    /   ← 中心节点 + 连线
   \  ╱              ╲  /
    ◇──────────────────◇     ← 盾形巢（安全）
```

**Logo 变体**：
- 主 Logo：几何猫面 + "狸知" 中文标（暗色背景白色图形）
- 图标 Logo：仅几何猫面（用于托盘、favicon）
- 水平 Logo：猫面 + "狸知知识库"（用于导航栏、官网）
- 英文 Logo：猫面 + "Lizhi Knowledge"（用于国际化场景）

### 2.3 字体系统

```
┌─────────────────────────────────────────────┐
│  UI 字体：Inter Variable                     │
│  ├── 中文回退：PingFang SC / Microsoft YaHei │
│  ├── 特征设置："cv01" "ss03" （几何感）      │
│  └── 用途：界面、导航、按钮、标题             │
├─────────────────────────────────────────────┤
│  代码字体：JetBrains Mono / Cascadia Code    │
│  ├── 等宽特征：连字 "liga"                   │
│  └── 用途：编辑器、代码块、技术标注           │
├─────────────────────────────────────────────┤
│  展示字体：Inter Display（大尺寸负间距）      │
│  └── 用途：Hero 标题、大字报、品牌宣传         │
└─────────────────────────────────────────────┘
```

**设计理念**：技术极客风格不需要衬线体的「人文温度」—— 猫系温度通过暖爪色注入，字体保持几何精确。

### 2.4 图标与插画

#### 图标

- 系统图标：**Lucide Icons**（已在使用，几何轮廓、线条一致）

### 2.5 动效语言

| 类型 | 描述 | 时长 | 缓动 |
|------|------|------|------|
| 微交互 | hover 颜色/阴影过渡 | 150ms | ease-out |
| 面板切换 | 侧栏展开/折叠 | 200ms | ease-in-out |
| 内容加载 | 骨架屏 shimmer | 1.2s | linear（循环） |
| 图谱力导向 | 节点位置过渡 | 300ms | spring-like |
| 锁定模糊 | vault 锁定模糊遮罩 | 150ms | ease-in |
| 暖爪脉冲 | CTA/通知徽标呼吸 | 2s | ease-in-out |

**守夜猫加载动画**：解锁时几何猫眼节点连线逐渐点亮（联结蓝），完成时瞳孔微微放大。

### 3.1 视觉主题与氛围

狸知知识库是一个**暗色工程终端风格的桌面应用**。整体氛围是精密、克制的技术工具，而非消费级产品。设计参考 Linear 的暗色深度叠加 + Supabase 的终端气质，但以「暖爪色」锚定猫系品牌的温度感。

**关键特征**：
- 暗色画布：`#141619` 主背景，`#1a1d23` 面板，`#23262d` 悬浮态

- 联结蓝 `#5b9fd4` 作为主交互色（选中、焦点、图谱）
- 暖爪色 `#d4a574` 作为唯一暖色锚点（CTA、品牌标记、通知）
- 六边形 + 菱形几何图形作为品牌装饰母题
- Inter Variable 字体，`"cv01"` `"ss03"` 几何特征

| `space-16` | 64px | 页面级间距 |

#### 圆角

| 令牌 | 值 | 用途 |
|------|-----|------|
| `radius-sm` | 4px | 小标签、代码块 |
| `radius-md` | 6px | 按钮、输入框、选中态 |
| `radius-lg` | 8px | 卡片、面板、菜单 |
| `radius-xl` | 12px | 模态框、大型面板 |
| `radius-full` | 9999px | 标签、徽标、pill 按钮 |

#### 安全区域

- 编辑器内边距：左右各 48px（专注模式 96px）
- 内容最大宽度：720px（编辑器正文），1200px（insights 仪表盘）
- 侧栏最小可交互宽度：48px

### 3.6 深度与层次

暗色主题下不使用传统阴影。深度通过背景色阶 + 边框透明度层级实现。

| 层级 | 背景 | 边框 | 场景 |
|------|------|------|------|
| L0 最深 | `#0d0f12` | 无 | 代码区、标题栏 |
| L1 画布 | `#141619` | 无 | 主工作区 |
| L2 面板 | `#1a1d23` | `rgba(255,255,255,0.04)` | 侧栏、卡片 |
| L3 悬浮 | `#23262d` | `rgba(255,255,255,0.08)` | 下拉菜单、tooltip |
| L4 模态 | `#1a1d23` | `rgba(255,255,255,0.12)` + 背板 `rgba(0,0,0,0.6)` | 对话框 |
| L5 焦点 | — | `2px solid rgba(91,159,212,0.3)` | 焦点环 |

### 3.7 设计守则

#### 应该做

- 保持暗色底色的一致层级 —— 永远从 `#141619` 起步
- 暖爪色仅用于品牌锚点（CTA、Logo、通知），一个界面 ≤ 3 处
- 联结蓝用于交互信号（链接、选中、图谱、焦点环）
- 使用半透明白色边框代替传统阴影 —— 暗色下阴影不可见
- Inter Variable 全局开启 `"cv01"` `"ss03"`
- 代码区使用 JetBrains Mono + 连字
- 圆角保持克制（4-8px 为主，标签用 full-pill）

#### 不应该做

- 不要使用纯白文字（`#ffffff`）→ 用 `#f0f1f2`
- 不要在暗色背景上使用 box-shadow
- 不要把暖爪色用作大面积背景或装饰色
- 不要使用 12px 以上的卡片圆角（除非模态框）
- 不要引入暖爪色之外的暖色调（橙/金/红语义色除外）
- 不要使用渐变按钮或卡片背景
- 不要使用 emoji 作为 UI 图标（除非加密状态指示器）
- 不要在 UI 中放置卡通猫图像 —— 几何猫 Logo 是唯一的猫系表达

### 3.8 响应式行为

桌面应用，窗口最小宽度 800px（侧栏折叠态 600px）。

| 窗口宽度 | 行为 |
|----------|------|
| ≥ 1200px | 三栏：侧栏(240) + 编辑区 + 反向链接面板(240) |
| 900-1200px | 两栏：侧栏(200) + 编辑区 |
| 600-900px | 单栏：侧栏折叠，可选展开浮层 |
| < 600px | 单栏紧凑（最小可用的底线） |
