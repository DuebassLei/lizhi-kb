## 1. 文档概述

### 1.1 产品愿景
狸知知识库致力于为用户提供一个 **100% 本地化、端到端加密、零网络依赖** 的个人加密知识库。我们不追求云端的广度协作，而是聚焦于个人知识的深度沉淀、联结与绝对的数据主权。

**Slogan**：你的加密知识库，猫一样安静。

### 1.2 v1.5 版本核心目标
在 v1.0（极致写作、热力图看板、安全导出）的基础上，v1.5 聚焦于 **“知识网络化”** 与 **“极致安全防御”** 两大维度：
1. **知识网络**：引入双向链接、局部关系图谱，打破文件夹层级，实现知识联结梳理。
2. **极致安全防御（本次重点）**：完善软件启动验证机制（App Lock）与全局/导出水印系统，防御物理偷窥、截屏泄露与暴力破解。

## 2. 全局安全与访问控制 (核心新增模块)

本模块是 狸知知识库 区别于市面常规笔记软件的“安全护城河”，必须在产品交互和底层技术上做到滴水不漏。

### 2.1 软件启动与身份验证 (App Lock & Vault Unlock)

**需求描述**：控制用户进入软件及解密知识库的权限，确保设备遗失或短暂离开时数据不被窥探。

#### 2.1.1 验证方式矩阵
| 验证方式 | 触发场景 | 技术实现原理 | 安全等级 |
| :--- | :--- | :--- | :--- |
| **主密码 (Master Password)** | 首次启动、生物识别失败、重启后 | 用户输入密码 -> 派生密钥 (Argon2id) -> 解密本地 SQLite 主库 | 最高 (核心基石) |
| **生物识别 (Biometrics)** | 日常解锁、超时锁定后唤醒 | 调用系统 API (Windows Hello / macOS Touch ID)，系统级安全飞地 (Secure Enclave) 验证通过后，释放内存中缓存的解密密钥 | 高 (便捷与安全的平衡) |
| **系统级 PIN/密码** | 无生物识别硬件时的备选 | 调用系统原生认证弹窗 (如 Windows Credential Manager) | 中 |

#### 2.1.2 自动锁定策略 (Auto-Lock)
- **闲置锁定**：支持设置无操作 `1分钟 / 5分钟 / 15分钟 / 30分钟 / 永不` 后自动锁定。锁定后界面模糊，需重新验证。
- **失焦锁定**：当应用窗口失去焦点（切换到其他应用）或最小化时，可选“立即锁定”。
- **休眠/锁屏联动**：监听到系统休眠或锁屏事件，**强制立即锁定**，并清空内存中的明文数据与索引。

#### 2.1.3 防暴力破解与“恐慌模式”
- **阶梯延迟**：连续输错密码 3 次，锁定 30 秒；错 5 次，锁定 5 分钟；错 10 次，锁定 24 小时。
- **诱饵库 (Plausible Deniability)**：若用户在设置中配置了“诱饵密码”，当遭遇胁迫输入诱饵密码时，软件将正常解锁并展示一个**预先准备的、包含无害内容的“诱饵知识库”**，从密码学上无法证明真实隐藏库的存在。

### 2.2 全局水印与防窥系统 (Watermark System)

**需求描述**：防止内部人员或旁人通过手机拍照、截屏、录屏等物理或软件手段泄露敏感内容，并支持泄露后的溯源。

#### 2.2.1 界面防窥水印 (Screen Watermark)
- **启用开关**：在“设置 -> 安全与隐私”中提供全局开关。
- **自定义内容**：支持动态变量组合，如 `${用户名} + ${设备MAC后4位} + ${当前时间}`。
- **样式配置**：
  - **透明度**：10% ~ 30%（默认 15%，确保不干扰阅读）。
  - **倾斜角度**：-15° 到 -45°（默认 -22°）。
  - **密度/间距**：紧凑、标准、稀疏。
  - **颜色**：跟随主题（浅色模式用浅灰，深色模式用深灰）。
- **覆盖范围**：编辑器、图谱、看板、设置页（密码输入框除外）。
- **防篡改**：水印层必须设置 `pointer-events: none`，且通过底层渲染或高 z-index 固定，防止通过 DOM 审查元素隐藏。

#### 2.2.2 导出水印 (Export Watermark)
- **强制/可选策略**：管理员或用户可设置“导出 PDF/图片时强制添加水印”或“每次导出时弹窗询问”。
- **样式独立**：导出水印的样式（字号、颜色、透明度）可与界面水印独立配置，确保打印或分享时的可读性。

#### 2.2.3 隐形盲水印 (Blind Watermark) [高级安全特性]
- **原理**：在导出的图片或 PDF 转换的图片中，通过频域算法（如 DCT 离散余弦变换）嵌入肉眼不可见的数字水印（如用户 ID 或设备指纹）。
- **用途**：当截图或导出的文件在外部泄露时，可通过 狸知知识库 内置的“盲水印提取工具”还原出泄露源头的设备信息。
- **性能约束**：仅在导出操作时异步计算，不影响日常编辑性能。

## 3. 核心业务需求 (v1.5 知识网络)

### 3.1 双向链接与反向链接 (Bi-directional Links)
- **创建与跳转**：输入 `[[` 唤起全局搜索，支持拼音/模糊匹配。`Ctrl/Cmd + Click` 跳转。
- **悬浮预览**：鼠标悬停 500ms 弹出只读预览卡片（前 200 字 + 首图）。
- **反向链接面板**：右侧边栏展示 `Linked Mentions`（已链接提及，带上下文）和 `Unlinked Mentions`（未链接提及，支持一键转为双链）。

### 3.2 局部关系图谱 (Local Graph View)
- **视图范围**：以当前文档为中心，向外延伸 2 个层级 (2-depth)。
- **节点视觉**：中心节点高亮，双链节点为主题色，附件节点为灰色方形。
- **交互**：力导向图布局，支持拖拽固定、滚轮缩放、单击预览、双击打开。
- **渲染**：SVG 力导向布局，支持拖拽固定、滚轮缩放、单击预览、双击打开。

### 3.3 编辑器、看板与安全导出
- **编辑器**：Markdown 双模式（源码/所见即所得），支持打字机滚动、专注模式。
- **看板热力图**：GitHub 风格写作热力图，双链操作均计入编辑次数。支持锁屏模糊。
- **安全导出**：支持 MD/PDF/HTML/DOCX/长图。导出时自动清洗元数据，支持正则敏感词过滤，图片自动打包 ZIP 或 Base64 内嵌。

## 4. 技术架构与开发指南 (TDD)

### 4.1 技术栈选型
| 层级 | 技术选型 | 选型理由 |
| :--- | :--- | :--- |
| **客户端框架** | **Tauri 2.0** (Rust + Webview) | 相比 Electron，内存占用极小，Rust 后端提供原生级安全与性能，完美支持系统级 API（如防截屏、生物识别）。 |
| **前端 UI** | **Vue 3 + Pinia + TailwindCSS** | Composition API，Pinia 管理 vault/文档/编辑器状态，Tailwind 便于水印与主题。 |
| **编辑器引擎** | **ProseMirror / TipTap** | 高度可定制的富文本/Markdown 底层，支持复杂的 Block 和双链插件扩展。 |
| **本地数据库** | **SQLite (SQLCipher)** | 业界标准的本地加密数据库，支持全文检索 (FTS5)。 |
| **图谱渲染** | **SVG + composable** | 局部关系图谱，支持节点拖拽与缩放。 |
| **加密算法** | **AES-256-GCM + Argon2id** | GCM 模式提供认证加密（防篡改），Argon2id 是目前抗 GPU 暴破最强的密码派生函数。 |

### 4.2 核心加密与解锁流程 (时序逻辑)

```text
[用户输入主密码] 
      │
      ▼
[Rust 后端: Argon2id 派生 KEK (Key Encryption Key)]
      │
      ▼
[使用 KEK 解密本地 Keychain 中的 DEK (Data Encryption Key)]
      │
      ▼
[将 DEK 载入内存，初始化 SQLCipher，挂载知识库]
      │
      ▼
[前端: 构建内存级倒排索引与双链图谱 (不落盘)]
      │
      ▼
[解锁成功，渲染主界面，启动闲置计时器]
```
*开发注意*：DEK 在内存中必须使用 `mlock` (macOS/Linux) 或 `VirtualLock` (Windows) 锁定，防止被交换到硬盘的 Swap 文件中。应用锁定时，必须主动覆写 (Zeroing) 该内存区域。

### 4.3 水印渲染技术方案

#### A. 界面防窥水印 (CSS + Canvas 混合方案)
- **实现**：在全局 Layout 最顶层覆盖一个 `position: fixed; z-index: 9999; pointer-events: none;` 的 Canvas 层。
- **绘制**：使用 Canvas API 绘制重复的文本图案，应用 `globalAlpha` 控制透明度，`rotate` 控制倾斜。
- **防绕过**：通过 Rust 后端定时向前端发送“心跳”，前端校验 Canvas 层是否被 DOM 篡改（如 `display: none` 或被移除），若检测到异常，立即触发“恐慌锁定”。

#### B. 导出盲水印 (频域算法)
- **实现**：在 Rust 后端集成 `rust-blind-watermark` 或类似库。
- **流程**：导出图片时 -> 将图片转为矩阵 -> 进行 DCT (离散余弦变换) -> 在特定频段嵌入用户 Hash -> 逆 DCT 生成最终图片。
- **提取**：提供独立的 CLI 工具或隐藏的设置页入口，输入疑似泄露图片，提取其中的 Hash 进行比对。

### 4.4 内存索引与性能优化
- **双链索引**：写入时增量更新 `links` 表；启动时校验 checksum。Rust 构建有向图经 Tauri Event 传递给前端 Pinia 存储。
- **全文搜索**：利用 SQLite 的 **FTS5** 扩展。由于数据库本身是加密的，FTS5 索引也是加密落盘的。查询时，SQLCipher 在内存中解密并匹配，速度可达毫秒级。支持拼音搜索需集成 `pinyin-rs` 进行分词扩展。

## 5. 数据与文件结构设计

### 5.1 本地目录结构
```text
~/.lizhi-kb/
├── config.enc              # 加密的全局配置（包含水印设置、锁定策略）
├── vault.db                # SQLCipher 主数据库（存储元数据、标签）
├── keys.enc                # 加密的密钥链（存储 DEK，需主密码解密）
├── workspace/              # 工作区（存放加密后的实体文件）
│   ├── inbox/
│   ├── projects/
│   └── assets/             # 加密的图片/附件资源
└── logs/
    └── audit.log           # 审计日志（导出、登录失败记录，明文但仅记录操作行为，不记录内容）
```

### 5.2 核心数据库表设计 (SQLite / SQLCipher)
```sql
-- 文档元数据表 (内容实体以 .md 加密文件形式存在 workspace 中)
CREATE TABLE documents (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    path TEXT UNIQUE NOT NULL,
    tags TEXT,              -- JSON array
    created_at INTEGER,
    updated_at INTEGER,
    word_count INTEGER
);

-- 双链关系表 (用于快速查询反向链接)
CREATE TABLE links (
    source_id TEXT,
    target_id TEXT,
    context_text TEXT,      -- 链接所在的上下文段落
    PRIMARY KEY (source_id, target_id)
);

-- 审计日志表
CREATE TABLE audit_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    action TEXT,            -- 'EXPORT', 'LOGIN_FAIL', 'LOCK'
    target_file TEXT,
    timestamp INTEGER,
    details TEXT
);
```

## 6. 非功能性需求 (NFR) 与测试标准

### 6.1 性能指标
- **冷启动**：10,000 篇文档规模下，从输入密码到主界面可交互 < 2.0 秒。
- **自动锁定响应**：系统休眠/失焦事件触发后，界面模糊并清空内存的延迟 < 100ms。
- **水印渲染**：界面防窥水印开启时，编辑器滚动的帧率下降 < 5%。

### 6.2 安全红线 (Security Red Lines)
1. **零网络请求**：应用在任何情况下（包括检查更新、崩溃报告）均不得发起未经用户显式授权的网络请求。可通过抓包工具 (Wireshark) 进行自动化测试拦截。
2. **内存零残留**：应用锁定或退出后，使用内存扫描工具 (如 Volatility) 扫描进程内存，不得残留主密码、DEK 密钥或文档明文。
3. **防截屏 API**：在 Windows 上调用 `SetWindowDisplayAffinity(hwnd, WDA_EXCLUDEFROMCAPTURE)`；在 macOS 上调用 `CGDisplayStreamCreate` 相关拦截或设置 `sharingType`，确保系统级截图工具截出的画面为全黑或模糊。

### 6.3 兼容性
- **操作系统**：Windows 10/11 (x64/ARM64), macOS 12+ (Intel/Apple Silicon), Ubuntu 22.04+。
- **硬件要求**：最低 4GB RAM，推荐 8GB 以上（用于支撑内存索引）。
