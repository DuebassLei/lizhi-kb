# 公众号「插入模块」说明清单

工作区切换到 **公众号** 预览后，顶栏点 **插入模块** 可插入排版组件。  
语法为 Markdown 容器块：`:::组件名` … `:::`，保存后右侧预览即时渲染，复制到微信公众平台可用。

---

## 怎么用

1. 打开文档 → 分栏预览 → 选「公众号」
2. 点预览顶栏 **插入模块**，选模板
3. 在左侧编辑器里改占位文案
4. 预览满意后「复制到微信」或导出 HTML

**推荐成稿骨架（自媒体）：**

```text
开场钩子 → 本期要点 → 正文（步骤/对比/重点）→ 误区或金句
→ 行动清单 → 互动提问 → 三连支持 / 行动号召
```

---

## 写作结构

| 组件 | 指令名 | 用途 | 适合放哪 |
|------|--------|------|----------|
| 开场钩子 | `:::lead` | 3 秒抓住注意力，抛出读者痛点或好奇点 | 文首、H1 后第一段前 |
| 本期要点 | `:::summary` | 编号摘要，扫一眼就知道能带走什么 | 文首导读 / 文末复盘 |
| 问答 FAQ | `:::faq` | 预判读者疑问，降低跳出 | 中段或文末 |
| 行动清单 | `:::checklist` | 读完就能勾选执行的 Todo | 干货文文末 |
| 行动号召 | `:::cta` | 强调下一步动作（关注、在看、转发） | 文末 |
| 作者说 | `:::author-note` | 私人语气编后语，拉近关系 | 文末、署名旁 |
| 互动提问 | `:::engage` | 抛一个具体问题，拉评论 | 文末 CTA 前 |
| 三连支持 | `:::support` | 「点赞·转发·推荐」水彩分享插件 | 全文最末 |

## 字段速查（常用）

多数组件支持 `title:` 等字段；用 `|` 分行/分列。示例如下。

**FAQ（问 \| 答）：**

```markdown
:::faq[常见问题]
值不值得做？ | 关心复利就值得。
需要什么基础？ | 会用基础工具即可。
:::
```

**误区纠正：**

```markdown
:::myth[先澄清一个误区]
myth: 写够多自然有读者
truth: 读者留下，是因为你稳定解决他的问题
:::
```

**三连支持（可改文案）：**

```markdown
:::support
title: 喜欢就支持一下吧！
subtitle: 你的每一次互动，都是我持续创作的动力～
bubble: 有你真好！
thanks: 谢谢你
like-tag: 点赞
like-desc: 喜欢就点个赞吧！感谢你的支持！
share-tag: 转发
share-desc: 分享给更多朋友，让美好一起传递！
rec-tag: 推荐
rec-desc: 推荐给身边的人，让更多人看到！
:::
```

**数据亮点（数值 \| 标签 \| 备注）：**

```markdown
:::stats[关键数字]
3× | 效率提升 | 改造后
80% | 可复用模板 | 减少重写
:::
```

## 选用清单（写前自检）

- [ ] 开篇有没有「钩子」？没有 → 插入 **开场钩子**
- [ ] 读者 10 秒内能否知道收获？没有 → **本期要点**
- [ ] 教程有没有可执行步骤？有 → **步骤** / **行动清单**
- [ ] 有没有要澄清的误解？有 → **误区纠正**
- [ ] 有没有一句想被截图的话？有 → **金句**
- [ ] 文末有没有引导互动？没有 → **互动提问** + **三连支持** 或 **行动号召**
- [ ] 提示/警告是否用错颜色语义？危险操作用 **危险**，小提示用 **提示**
