# AI 写作小助手 · 产品设计

> **版本**: 1.0.0  
> **日期**: 2026-07-15  
> **状态**: 待实现  
> **关联**: [AI 助手](./2026-07-08-lizhi-ai-chat-design.md)、[知识卡片](./2026-07-15-knowledge-cards-design.md)、[资产库](./2026-07-14-asset-library-browse-design.md)

---

## 1. 目标

在编辑器内提供**独立弹窗向导**，面向**公众号 / 对外长文**，跑通完整流水线：

选题 → 大纲 → 正文 → 去 AI 味 →（可选）配图 →（可选）封面 → 定稿入知识库

复用现有 AI 配置与流式 LLM（`aiService` / Rust `ai/*`），**不**与右栏 AI 助手会话合并，**不**并入 Agent 工作台。

### 1.1 成功标准

1. 编辑器工具栏可打开写作助手弹窗；AI 未启用时引导去设置。
2. 大纲格式、写作风格（预设 + 补充）、目标字数、去 AI 味强度、参考知识库、**启用封面**、**启用配图**均可配置；默认值可持久化。
3. 可从任一步切入；每步「采用」门禁；支持本步重新生成与手工编辑。
4. 配图：按章节生成**结构化图示提示词** → 用户可改 → **Canvas 版式卡**落盘；非 AI 文生图。
5. 封面：模板合成或上传；与配图开关独立。
6. 定稿可选「新建文档」（默认）或「写入当前」；Markdown 含 `asset://` 图引用，预览可见。
7. UI 明确「版式配图，非摄影 / 文生图」；不做微信草稿箱发布。

### 1.2 非目标（本期不做）

| 项 | 说明 |
|----|------|
| AI 文生图 | DALL·E 等；提示词不送扩散模型 |
| 微信草稿箱 / 一键发布 | 二期 |
| 克隆某篇旧文风格 | 仅预设 + 自由补充 |
| 多场次历史列表 | 仅单会话草稿 |
| Inline ghost 补全 | 与本助手无关 |
| 并入 ChatPanel / CC Workbench | 并列隔离 |

## 2. 架构

```
MarkdownEditorToolbar
  → WritingAssistantDialog
       → writingAssistant store（会话 / 门禁 / stale）
       → writingAssistantService → aiService → Tauri ai_*（文本步骤）
       → WaCanvasRenderer → Blob → save_asset（封面 / 配图）
       → documents.create | 替换当前 content（定稿）
```

| 边界 | 约定 |
|------|------|
| vs AI Chat | 独立 store；不写 Chat 历史；文案标明「写作助手」 |
| vs 笔记助手 `writeEnabled` | 定稿为用户显式写库，不依赖 Agent 写开关 |
| vs 知识卡片 | 卡片是「已有正文 → 分页贴图」；本助手是「生成长文 + 版式配图」；模板体系可不共用，避免拖垮依赖 |

---

## 3. 信息架构与入口

| 项 | 说明 |
|----|------|
| 入口 | 编辑器工具栏「写作助手」；快捷键可后置 |
| 壳 | 大尺寸 Modal（非窄 `InputDialog`）；顶栏步骤条 |
| 挂载 | `EditorPane` / Workspace 层；不新开路由 |
| 关窗 | 保留单会话草稿；「放弃」清空 |
| 锁定 vault | 清空草稿与会话（与 Chat 一致） |

### 3.1 步骤顺序

1. **选题** — 意图输入；可选「参考知识库」；候选标题/角度 → 采用  
2. **大纲** — 按大纲格式配置生成可编辑树 → 采用  
3. **正文** — 按风格 / 字数流式生成 → 采用  
4. **去 AI 味** — 按强度改写；可对比 → 采用  
5. **配图**（若启用）— 结构化提示词 → 可编辑 → Canvas / 上传  
6. **封面**（若启用）— 模板 / 上传  
7. **定稿** — 预览全文；新建或写入当前  

关闭的步骤在进度条隐藏或标「已跳过」。可跳到**已到达**步骤；可从任一步「切入」开始（跳过上游则上游留空或仅用用户粘贴内容）。

### 3.2 下游失效（stale）

修改已采用的上游（选题 / 大纲 / 正文）后，下游步骤标记 `stale`，CTA「建议重生成」；**不**自动清空下游，避免误删。

## 4. 配置

会话内可改。默认值持久化优先级：扩展 `ai-config.json` 的 `writingAssistant` 对象（经 `get_ai_config` / `set_ai_config`）；若本期改 Rust 配置面过大，可先把默认值放在 `vault-ui-state` 的 `writingAssistantDefaults`，与草稿分开，实现计划里二选一并写死。

| 配置 | 作用于 | v1 形态 |
|------|--------|---------|
| 大纲格式 | 大纲 | 预设：`H2/H3`、`一/（一）/1.`、`纯列表`；可选自定义说明 |
| 写作风格 | 正文、去 AI 味 | 预设：`科普清晰`、`故事感`、`严谨论述`、`轻松口语` + 风格补充文案 |
| 目标字数 | 正文 | 短 / 中 / 长 或大致数字 |
| 去 AI 味强度 | 去 AI 味 | 轻 / 中 / 重 |
| 参考知识库 | 选题、正文（及提示词上下文） | 布尔；开则走 RAG 片段进 prompt |
| 启用封面 | 封面步 | 布尔；与配图独立 |
| 启用配图 | 配图步 | 布尔；与封面独立 |
| 默认封面模板 | 封面 | 模板 id |
| 默认配图版式 | 配图 | 版式 id |

改配置只影响**之后**的生成；已采用内容需「重新生成」才更新。

## 5. 配图与封面（Canvas）

### 5.1 能力预期

本地 **Canvas（或 SVG → Canvas）版式卡**：标题、说明、色板、几何/图标装饰。  
**不是**摄影级或文生图像素还原。UI 与定稿说明须写明。

### 5.2 配图：提示词 → Canvas

在正文采用后（或配图步触发）：

1. LLM 按大纲章节输出**结构化**提示词（JSON），禁止仅依赖自由长句驱动像素。  
2. 用户可改字段后「按此重绘」；单节可关闭不出图。  
3. `WaCanvasRenderer` 映射 `layout` / `mood` / `keywords` → 模板与装饰 → PNG → `save_asset`。  
4. 可选：用户上传本地图覆盖该节。

**结构化字段（契约）**

| 字段 | 类型 | 说明 |
|------|------|------|
| `sectionId` | string | 对应大纲节点 |
| `title` | string | 图上主文案 |
| `caption` | string | 一句画面说明 |
| `keywords` | string[] | 视觉关键词（有限词表优先） |
| `layout` | `hero` \| `split` \| `bullets` | 版式 |
| `mood` | `cool` \| `warm` \| `neutral` | 色板 |
| `notes` | string? | 自由备注，**仅展示**，不直接驱动像素 |

### 5.3 封面

- 模板合成：标题 / 副标题 / 风格色（可从选题标题带出）。  
- 或上传本地图。  
- 与配图共用渲染基础设施，尺寸比例不同（封面偏公众号封面比，配图偏文内宽图/方图；具体比例实现时定，须可配置常量）。

### 5.4 模板下限

- 封面模板 **≥ 3**  
- 配图版式 **≥ 3**（与 `layout` 枚举对齐或为其皮肤变体）  
- `keywords` → 装饰为有限映射表，未知词回退默认装饰

### 5.5 Markdown 约定

- 正文内配图：`![caption](asset://<id>)`，插在对应 `##` 节下（与现有 `ASSET_PREFIX` / 预览 `resolveAssetUrl` 一致）。  
- 封面：YAML frontmatter 固定为 `cover: asset://<id>`（与正文配图同一 `ASSET_PREFIX`；预览若不读 frontmatter，定稿组装时可同时在标题下插入一张封面图，避免「有字段看不见」）。  
- 文首可加简短 HTML 注释：`<!-- 配图为本地版式卡，非 AI 文生图 -->`。

## 6. 定稿

| 选项 | 行为 |
|------|------|
| 新建文档（默认） | `documents` 创建并打开；当前文档不变 |
| 写入当前 | 替换当前 `content`（覆盖前确认对话框） |

组装顺序：frontmatter（含 cover）→ 标题 → 各节正文与配图。  
过程稿仅在写作会话中；定稿前不自动写库。

## 8. 安全

- API Key / 出站请求仅 Rust；前端不持久化明文 Key。  
- `network_gate` 与现有 AI 一致；启用参考知识库时提示：检索片段可能发往当前供应商。  
- 解锁会话内处理正文与草稿；锁定清空。  
- 配图 / 封面二进制走 `save_asset`（加密库下随 vault 资产策略），禁止临时写到仓库目录或未加密公共路径。  
- 定稿写文档走正规文档 API，日志不含正文全量与 Key。

---

## 9. 错误与取消

- 各文本步骤支持取消流式。  
- 单张 Canvas 或 `save_asset` 失败：可重试该张，不阻断其他图与定稿（该节可无图或占位）。  
- AI stub / 非 Tauri 预览：与现有 AI 行为一致并提示。  
- 结构化提示词解析失败：保留原始文本供手改，或一键「重新生成提示词」。

## 12. 已知局限（接受）

- Canvas 无法表达任意自然语言画面；结构化字段 + 有限装饰是刻意约束。  
- 去 AI 味降低痕迹，不保证无人能辨。  
- 多次 LLM 调用有成本；靠跳步与取消控制。  
- 桌面宽屏优先；窄屏可将 Modal 拉满。

## 13. UI / UX 设计规范

> 对齐 [品牌设计书](../../brand/lizhi-brand-design.md)：墨巢暗色工程 + 暖爪点缀；不引入紫渐变 / 奶油衬线 / 多阴影卡片堆。  
> 交互对齐现有 `ConfirmDialog` / 知识卡片 Dialog：`Teleport` + 遮罩 blur + `rounded-xl` + token 色。

### 13.1 设计原则（本弹窗）

| 原则 | 落地 |
|------|------|
| 一条主路径 | 顶栏步骤条是唯一导航；一屏只做一个主任务 |
| 暖爪节制 | 全窗最多 2～3 处暖爪：主 CTA、当前步骤指示、流式进行中徽标 |
| 联结蓝作结构 | 焦点环、选中大纲节点、链接、次要强调用联结蓝，不用暖爪铺满 |
| 8px 栅格 | 间距 / padding 仅用 8、12、16、24、32 |
| 反馈必达 | 生成中、成功采用、stale、失败均可感知（文案 + 色 + 可选短动效） |
| 可访问 | 对比度 ≥ 4.5:1；`Esc` 关闭（有未采用变更时先确认）；焦点陷阱在 Modal 内；步骤条可键盘左右 |

### 13.2 视觉令牌（复用 CSS 变量，不新建色板）

| 用途 | Token |
|------|------|
| 遮罩 | `bg-overlay` + `backdrop-blur-sm` |
| 面板 | `bg-surface-1` / `var(--color-panel)`，边框 `border-border-strong` |
| 主文 | `var(--color-text)` |
| 辅文 / 能力声明 | `text-muted` / `var(--color-muted)` |
| 主 CTA | 暖爪：`var(--color-paw)` 底 + 深文字或反白（以现有 `Btn` primary 为准） |
| 次按钮 | ghost / surface-2 |
| 破坏（放弃） | 错误红描边或文字，非实心大红块 |
| 成功（已采用） | `var(--color-hold)` / 成功绿勾 |
| stale 警告 | 告警黄点 + 「建议重生成」 |
| 焦点 | 联结蓝 2px ring（`focus-ring`） |

**禁止**：弹窗内紫蓝渐变背景、大面积 glow、圆角药丸标签堆、多层级 card 套 card。

### 13.3 布局结构

桌面默认 **960×720** 可视区域（`max-w-5xl`，`max-h-[min(720px,85vh)]`）；窄于 768px：**全屏近边**（`inset-3` 或全屏）。

```
┌─────────────────────────────────────────────────────────────┐
│ 顶栏 56px：标题「写作助手」· 能力 chip · 配置 · 放弃 · 关闭   │
├─────────────────────────────────────────────────────────────┤
│ 步骤条 48px：①选题 ②大纲 ③正文 ④去AI味 [⑤配图] [⑥封面] ⑦定稿 │
├──────────────────────────────┬──────────────────────────────┤
│                              │                              │
│  主区（当前步编辑）           │  侧鉴（可选，≥960px）         │
│  flex-1 · overflow-auto      │  264px：配置摘要 / 预览缩略   │
│  padding 24                  │  封面·配图步改为大预览为主    │
│                              │                              │
├──────────────────────────────┴──────────────────────────────┤
│ 底栏 64px：次要（重新生成） · 状态文案 · 主 CTA（采用/下一步） │
└─────────────────────────────────────────────────────────────┘
```

- **配图 / 封面步**：主区改为「左表单（提示词字段）| 右 Canvas 预览」，侧鉴可收起。  
- **定稿步**：主区全文 Markdown 只读预览 + 落点单选（新建 / 写入当前）。

### 13.4 顶栏

- 左：Lucide `PenLine` 或 `Sparkles`（16px，联结蓝）+ 标题 18px/590「写作助手」  
- 中左：单个 muted chip「公众号长文 · 版式配图」——能力预期，避免误解文生图  
- 右：`配置`（齿轮，打开 `WaConfigPanel` 抽屉/弹出层）· `放弃`（文字按钮）· `关闭`（图标）  
- 无第二行营销文案；副说明只放 chip

### 13.5 步骤条

- 水平可滚动；每步：序号圆 + 短标签  
- **状态**：`todo` 弱文 / `current` 暖爪底或暖爪环 + 主文 / `done` 成功绿勾 / `skipped` 删除线弱文 / `stale` 黄点  
- 仅 `done` 与 `current` 及「已切入可达」可点击；动画 150ms 色变，勿弹跳放大  
- 关闭的封面/配图不占位，步骤自动重编号视觉（逻辑 id 不变）

### 13.6 分步主区模式

| 步骤 | 主区形态 | 主 CTA |
|------|----------|--------|
| 选题 | 大文本域（意图）+ 开关「参考知识库」+ 候选标题卡片列表（单选） | 采用此选题 |
| 大纲 | 可编辑树/缩进列表；工具条「按格式重生成」 | 采用大纲 |
| 正文 | 流式 Markdown 编辑区（等宽可选）；字数角标 | 采用正文 |
| 去 AI 味 | 左右或上下对比（原稿 \| 新稿）；强度已在配置 | 采用去味稿 |
| 配图 | 章节列表 + 结构化字段表单 + Canvas 实时/按钮刷新预览 | 采用配图 |
| 封面 | 模板三选一缩略图 + 标题/副标输入 + 大预览；上传入口次要 | 采用封面 |
| 定稿 | 预览 + 落点 radio + 风险提示（写入当前） | 定稿写入 |

**空态**：几何简笔（菱形节点即可），一句说明 + 主生成按钮；不放插画海。

**生成中**：主区顶部细进度条（联结蓝）或暖爪呼吸点；主 CTA 变为「停止」；输入只读。

### 13.7 配置面板

- 自右侧滑入抽屉（宽 320px）或顶栏下 popover（实现择一，优先抽屉以免挡主 CTA）  
- 分组：`写作`（格式/风格/字数/去味）· `图`（启用封面、启用配图、默认模板）· `检索`（参考知识库）  
- 控件：原生感 segment / select / textarea；风格补充为多行  
- 底部：「恢复默认」文字链；改动立即写入会话配置，不另「保存」

### 13.8 配图 / 封面预览

- 预览区深底（墨巢黑）+ 细边框；图卡阴影一层即可（`shadow-float` 或更轻）  
- 「按此重绘」次按钮在预览上方；失败 toast/行内错误红字  
- 上传：虚线框 dropzone，文案「或使用本地图片」

### 13.9 底栏与按钮层级

| 优先级 | 控件 | 样式 |
|--------|------|------|
| 主 | 采用 / 定稿 | `Btn` primary（暖爪） |
| 次 | 重新生成 | ghost |
| 危 | 停止（仅流式） | 描边 |
| 状态 | 左下或中：「已采用」「建议重生成」「未启用 AI」 | 12px muted / 告警色 |

同一底栏不超过 **一个** 实心主按钮。

### 13.10 动效（克制）

| 事件 | 动效 | 时长 |
|------|------|------|
| 开/关弹窗 | opacity + 轻微 translateY(8px) | 200ms ease-out |
| 步骤切换 | 主区内容 fade | 150ms |
| 流式 token | 无需逐字动画；光标/尾点即可 | — |
| 采用成功 | 步骤勾显现 | 150ms |
| stale 出现 | 黄点 fade-in | 150ms |
| 暖爪 CTA hover | 颜色至 paw 深色 | 150ms |

不使用整页 parallax、大弹簧、连续粒子。

### 13.11 文案语气

- 冷静短句：「采用大纲」「按此重绘」「写入会覆盖当前文档」  
- 能力声明固定句：「配图与封面为本地 Canvas 版式卡，非 AI 文生图」  
- 禁用夸张：「一键爆款」「智能魔法」等

### 13.12 实现落点（样式）

- 专用样式表：`src/styles/writing-assistant.css`（BEM：`wa-dialog`、`wa-steps`、`wa-pane`…）  
- 组件用现有 `Btn`、`focus-ring`、Lucide；颜色只引用 CSS 变量  
- `data-testid`：`wa-dialog`、`wa-step-<id>`、`wa-cta-adopt`、`wa-config` 等，便于 E2E

### 13.13 UI 验收

1. 暗色下正文/按钮对比度合格；焦点环可见。  
2. 暖爪出现次数符合 §13.1。  
3. 关封面+配图时步骤条不留空洞，主路径仍清晰。  
4. 768px 宽度下可完成选题→定稿（无横向裁切主 CTA）。  
5. 与 AI Chat 面板并排打开时，写作弹窗 z-index 高于编辑器附件、低于或等于全局 Confirm（写入确认用 ConfirmDialog）。
