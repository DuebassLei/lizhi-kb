## 1. 目标

防止知识库 Markdown 中的账号密码等敏感内容进入任何大模型上下文；同时保证编辑体验可用：源码可见、预览有提示、公众号导出不残留。

**核心用户场景**：

- 运维笔记里夹着测试/生产账号密码，仍要用 RAG / 笔记助手 / MCP / 工作台问其他问题
- 整篇客户资料永不进 AI：一键「禁止喂 AI」
- 从编辑栏一键插入 / 包裹 `:::ai-private`；「插入模块」亦提供同名模板，语法与现有 `:::module` 一致

**明确不做（v1）**：

- 「仍要发送」单次绕过
- 纯自动扫描当硬闸（无显式标记则不保证拦截）
- 自动改密文 / 密码本与笔记双链
- 围栏嵌套

---

## 2. 原则

1. **显式硬闸**：整篇 `ai_exclude` + 篇内 `:::ai-private`；未标记不承诺拦截  
2. **扫描是安全带**：启发式只提示，可一键包进围栏或标整篇；扫描本身不是闸  
3. **无发送例外**：已标记内容对 AI 出口一律不可见  
4. **一处执法**：Rust 统一 `sanitize_for_ai`；RAG / Agent / MCP / 工作台打开笔记共用  
5. **语法与插入模块对齐**：`:::` 围栏；「插入模块」提供「隐藏信息」模板  
6. **预览有提示、导出无占位**：预览替换为提示卡；公众号导出整段删除且不留占位

## 3. 标记模型

### 3.1 整篇：`ai_exclude`

| 项 | 约定 |
|----|------|
| 存储 | `documents.ai_exclude` INTEGER NOT NULL DEFAULT 0（迁移追加） |
| UI | 文档菜单 / 编辑器：「禁止喂 AI」；开启时标题旁小锁 |
| AI 行为 | 不参与 AI 检索；读正文返回明确中文错误（如「该笔记已禁止提供给 AI」）；列表在 AI 路径可过滤 |
| 非 AI | 编辑、搜索（应用内用户搜索）、图谱等照常；本字段不阻止用户本人阅读 |

`DocumentMeta` / IPC 需暴露 `aiExclude: boolean`（camelCase）。

### 3.2 篇内：`:::ai-private`

语法与微信排版模块一致：

```markdown
:::ai-private
用户名: admin
密码: s3cret
:::
```

| 项 | 约定 |
|----|------|
| 名称 | `ai-private`（固定） |
| 嵌套 | v1 不支持 |
| 未闭合 | 从 `:::ai-private` 起至文末均视为敏感（AI / 预览 / 导出一致，保守） |
| 源码 | 编辑器中原文可见、可编辑 |

### 3.3 半自动扫描

| 项 | 约定 |
|----|------|
| 触发 | 保存后，或即将组装 AI 上下文前（实现可选其一或两者；至少保存后） |
| 规则 | 启发式：含 `密码` / `password` / `secret` / `token` 等 + 赋值形态；且命中片段不在已有 `:::ai-private` 内 |
| 动作 | 提示条：「发现疑似敏感内容」→「包进隐藏信息」（能定位命中行则包裹该段，否则在光标处插入空模板）/ 「整篇禁止喂 AI」 / 「忽略」 |
| 忽略 | 按文档记一次（session 或轻量本地状态），避免刷屏 |
| 硬闸 | 扫描结果**不**单独构成拦截；用户未采纳则仍可能进模型（符合「显式硬闸」） |

## 4. 各出口行为

| 出口 | `ai_exclude` 整篇 | `:::ai-private` 块 |
|------|-------------------|-------------------|
| RAG / Agent 工具读 / MCP 读搜 / 工作台打开笔记 | 整篇不进模型 | 块内容删除后再送出 |
| 编辑器源码 | 照常 | 照常 |
| GFM / 公众号**预览** | — | 替换为提示卡，**不**含块内原文 |
| 公众号**导出** | — | 整段删除，**无占位**，尽量不留多余空行 |
| 密码本 `credential_entries` | 不在本文档范围；现状：不进 MCP/RAG，保持 | — |

预览提示卡文案（中文，可微调）：

> 🔒 已隐藏敏感内容 · 导出与 AI 均不可见

## 5. 架构与数据流

```
源码 Markdown（含 :::ai-private / ai_exclude）
        │
        ├─► 编辑器：原文可见；插入模块可插「隐藏信息」
        │
        ├─► 预览（GFM / 公众号预览）
        │     识别围栏 → 提示卡（无密文）
        │
        ├─► 公众号导出
        │     删除整块 :::ai-private（无占位）
        │
        └─► AI 出口
              Rust sanitize_for_ai(meta, content)
                · ai_exclude → 跳过 / 错误
                · 否则剥掉 :::ai-private 后再组 prompt 或返回给 MCP
```

### 5.1 落点

| 层 | 职责 |
|----|------|
| DB 迁移 | `documents.ai_exclude` |
| Rust 新模块（建议 `ai_privacy`） | 解析围栏、`sanitize_for_ai`、与 search/read/RAG/Agent 挂钩 |
| FTS | `fts_format_version` ≥ 4：索引 body 为 sanitize 后正文；`ai_exclude` 文档 body 为空；切换开关时重索引 |
| MCP | 读/搜走 `*_for_ai`；`GET /documents` **不返回** `ai_exclude` 文档 |
| `moduleSnippets` + `WechatModuleSelect` | 新增「隐藏信息」插入项与迷你预览 |
| `MarkdownEditorToolbar` + `markdownInsert` | 编辑栏「隐藏信息」快插按钮（见 §6.1） |
| `parseLayoutMarkdown` / 预览渲染 | `ai-private` → 提示卡 |
| 导出管线 | `ai-private` → 删除无占位 |

### 5.2 插入模块条目

| 项 | 值 |
|----|----|
| id | `ai-private` |
| 标题 | 隐藏信息 |
| hint | 账号密码等；预览提示、导出删除、不进 AI |
| group | **安全**（新建 `LAYOUT_MODULE_GROUPS` 项，避免与排版模块混） |
| kind | 专用（如 `privacy`），不走公众号装饰样式 |
| snippet | 见下 |
| 菜单迷你预览 | 锁图标 +「隐藏信息」，不展示示例密文 |

```markdown
:::ai-private
账号：
密码：
:::

```

### 5.3 MCP / 工作台错误

- 请求 `ai_exclude` 文档：返回明确中文错误（内部可读错误码可选，如 `AI_EXCLUDE`）  
- 含围栏的可读文档：返回内容已剥离敏感块；调用方无需知悉原文

## 6. UI 细节

- **禁止喂 AI**：文档菜单开关；与 `ai_exclude` 双向同步  
- **小锁**：开启时在标题/树节点旁显示（样式跟现有 muted 图标语言）  
- **扫描提示条**：非模态、可关闭；三动作见 §3.3  
- **用户文案**：一律中文  

### 6.1 编辑栏快速插入（必做）

落点：`MarkdownEditorToolbar`（与双链 / 链接同组或紧随其后加分隔组）。

| 项 | 约定 |
|----|------|
| 控件 | `BtnIcon`，`label` / `aria-label`：「隐藏信息」；`title`：「插入 :::ai-private（有选区则包裹）」 |
| `data-testid` | `editor-toolbar-ai-private` |
| 图标 | 锁（如 Lucide `Lock` / `Shield`），与品牌克制风格一致 |
| 无选区 | 在光标插入与 §5.2 相同的空模板，光标落在围栏体内（「账号：」行或首个可填行） |
| 有选区 | 用 `:::ai-private\n` + 选区 + `\n:::` 包裹；保留选区内容 |
| 共用 | 插入正文与「插入模块」snippet **同一常量**（避免两处文案漂移）；建议 `markdownInsert.insertAiPrivate`（或从 `moduleSnippets` 导出 snippet 再封装） |

与「插入模块 → 隐藏信息」并存：编辑栏是高频快路径，模块菜单是发现/学习路径。

## 7. 验收标准

- [ ] `ai_exclude=1` 的文档：RAG 不引用、Agent/MCP 读失败或跳过、工作台打开该笔记不注入正文  
- [ ] 含 `:::ai-private` 的文档：上述 AI 出口中块内明文不可出现  
- [ ] 未闭合围栏：从起始标记到文末均不进 AI / 不进导出正文  
- [ ] GFM / 公众号预览：显示提示卡，卡片内无密文  
- [ ] 公众号导出：无 `:::ai-private`、无提示占位、无明显多余空行块  
- [ ] 「插入模块」可插入「隐藏信息」，迷你预览无示例密码  
- [ ] **编辑栏**有「隐藏信息」按钮：无选区插入模板、有选区包裹；`data-testid=editor-toolbar-ai-private`  
- [ ] 密码本表仍不可经 MCP/RAG 读出  
- [ ] 无「仍要发送」入口  
- [ ] `pnpm verify` 通过

## 9. 分阶段

| 阶段 | 交付 |
|------|------|
| P0 | DB `ai_exclude` + Rust sanitize + RAG/Agent/MCP/工作台挂钩 + 文档开关 |
| P1 | `:::ai-private` 解析全出口 + 插入模块「隐藏信息」+ **编辑栏快插** + 预览提示卡 + 导出删除 |
| P2 | 半自动扫描提示条 |

建议 **P0→P1 同一切片优先**；P2 已实现：保存后 / 打开文档时启发式提示（包进围栏 / 整篇禁止 / 忽略）。

## 10. 风险与边界

- **假安全感**：未标记内容仍可能进模型；设置/AI 面板需短提示：「仅对已标记内容硬拦截」  
- **启发式误报**：故扫描非硬闸  
- **工作台 project 模式**：若 Agent 用 Bash/读盘绕过 lizhi-mcp，不在本护栏内；vault 模式 + lizhi-mcp 为默认防护面。Spec 声明：本功能覆盖「经狸知文档 API / 打开笔记注入」路径，不声称拦截任意 shell 读文件  
- **备份/同步**：`ai_exclude` 随 DB；围栏随 Markdown 正文，一并进备份
